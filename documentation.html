Layout

Five reasons why my site adheres to the principles of usable website design

  1. No extreme colours (Example Here)
  2. Constistency Accross pages. You're not going to get a shock when you hit the next page
  3. I used a web friendly font, and in my content paragraphs I modified the spacing to make it easier to read. Besides, You can't go wrong with Helvetica
  4. Pages are kept simple.
  5. PDF files can be nasty, and you come accross them a lot in research. PDF links get a warning added with CSS automatically not valid link The links to pdf files also support a 'data-size' not valid link

Effects

I used jQuery to automatically populate my navigation selection menus. This isn't really an effect but it allows you to move the window to a smaller dimension and the navigation to not get all bunched together. It also keeps the navigation menu synced in the header and in the footer. Ideally I would have set the current page in navigation class to active, but I didn't want to deal with regex too much, I would perfer to do that serverside on page generation. I suppose I could quickly add a hover function to do something to be of an "effect" but CSS is better for that. I've added the click on site name and return to home via jquery

I used jQuery to always display the page title in the sub header. I used css to toggle the displaying of the top navigation menu when the screen is smaller than a certain size, but I used jQuery to populate the selection options that are displayed. I'll add shadowbox or some other plugin for the hobbie page with photos. I'm randomly setting the background every minute via javascript and jQuery. On the about me page, I calculate my age with jQuery.

SEO

  1. Manually submit site to Google crawler
  2. Unique accurate summaries in the page description and title tags
  3. Simple to understand URL (www.amoriarty.com)
  4. Short urls paths and words not numbers and information the user doesn't need for filenames. *
  5. This site is too small currently, but I've already added the breadcrumb in the page header and would theoretically optimize my directory structure for easy navigation by both users and search engines.

Accessibility

  1. Three Flashes (2.3.2): Web pages do not contain anything that flashes more than three times in any one second period. (Level AAA)
  2. Multiple Ways (2.4.5): More than one way is available to locate a Web page within a set of Web pages except where the Web Page is the result of, or a step in, a process. (Level AA) navigation in header and footer
  3. Location (2.4.8): Information about the user's location within a set of Web pages is available. (Level AAA) breadcrumbs
  4. Consistent Navigation (3.2.3): Navigational mechanisms that are repeated on multiple Web pages within a set of Web pages occur in the same relative order each time they are repeated, unless a change is initiated by the user. (Level AA) This is similar to 2.4.5 but I thought I would mention again that my navigation menu is only set once in the header, and is the same on each page, the selection menu in the footer and header are both generated with javascript to ensure they remain the same.
  5. Use of Color (1.4.1): Color is not used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element. (Level A) Most importantly, it does not look like this
