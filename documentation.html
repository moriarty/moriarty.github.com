Layout

Five reasons why my site adheres to the principles of usable website design

  1. No extreme colours (Example Here)
  2. Constistency Accross pages. You're not going to get a shock when you hit the next page
  3. I used a web friendly font. You can't go wrong with Helvetica
  4. Pages are kept simple.
  5. PDF files can be nasty, and you come accross them alot in research. PDF links get a warning added with CSS automatically not valid link

Effects

I used jQuery to automatically populate my navigation selection menus. This isn't really an effect but it allows you to move the window to a smaller dimension and the navigation to not get all bunched together. It also keeps the navigation menu synced in the header and in the footer. Ideally I would have set the current page in navigation class to active, but I didn't want to deal with regex too much, I would perfer to do that serverside on page generation. I suppose I could quickly add a hover function to do something to be of an "effect"

I used jQuery to always display the page title in the sub header. I used css to toggle the displaying of the top navigation menu when the screen is smaller than a certain size, but I used jQuery to populate the selection options that are displayed. I'll add shadowbox for the page with photos. I've added a changing background image on the Home page via jQuery
